## SOP Template

When creating a new SOP, use this structure:

```markdown
# SOP: [Procedure Name]

## Purpose
Why this procedure exists.

## Scope
When to use this procedure.

## Prerequisites
- What's needed before starting

## Procedure

### Step 1: [Step Name]
Detailed instructions.

### Step 2: [Step Name]
Detailed instructions.

## Verification
How to confirm the procedure was successful.

## Troubleshooting
Common issues and solutions.

## Related
Links to related documentation.

## Revision History
| Date | Author | Changes |
|------|--------|---------|
| YYYY-MM-DD | Name | Initial version |
```

## Best Practices for SOPs

1. **Keep it simple** - Anyone should be able to follow
2. **Be specific** - Include exact commands, paths, values
3. **Include screenshots** - When helpful for UI procedures
4. **Test procedures** - Verify steps work before publishing
5. **Update regularly** - Keep in sync with actual processes
6. **Version control** - Track changes to procedures
