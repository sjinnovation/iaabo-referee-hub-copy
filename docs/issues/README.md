# Issue Tracking

## Overview

This directory contains documentation for tracking issues, bugs, and technical debt in IAABO Referee Hub.

## Issue Categories

### Bugs
Runtime errors, incorrect behavior, or broken functionality.

### Technical Debt
Code that needs refactoring, optimization, or cleanup.

### Improvements
Enhancements to existing functionality.

### Documentation
Missing or outdated documentation.

## Issue Template

When documenting an issue, use this structure:

```markdown
# Issue: [Short Title]

## Type
[Bug | Technical Debt | Improvement | Documentation]

## Priority
[Critical | High | Medium | Low]

## Description
Detailed description of the issue.

## Steps to Reproduce (for bugs)
1. Step 1
2. Step 2
3. ...

## Expected Behavior
What should happen.

## Actual Behavior
What actually happens.

## Impact
Who/what is affected and how severely.

## Proposed Solution
How to fix this issue.

## Related
- Links to related code, docs, or issues

## Status
[Open | In Progress | Resolved | Won't Fix]
```

## Issue Lifecycle

1. **Identify**: Document the issue
2. **Triage**: Assign priority
3. **Plan**: Schedule for resolution
4. **Resolve**: Fix and verify
5. **Close**: Document resolution

## Best Practices

1. **Be specific** - Include exact error messages, steps to reproduce
2. **Prioritize ruthlessly** - Not everything is critical
3. **Link related items** - Connect to features, PRs, other issues
4. **Update status** - Keep documentation current
5. **Document resolutions** - Help future debugging

## GitHub Issues

For active development, also use GitHub Issues. This directory is for:
- Long-term tracking
- Technical debt documentation
- Issues that span multiple PRs
- Historical reference
